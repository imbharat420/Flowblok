# Flowblok OS — Minimal System Design

Design a world-class product experience inspired by Linear, Raycast, Vercel, Framer, Arc Browser, and modern Apple software.

The interface must feel:

- Fast
- Quiet
- Technical
- Premium
- Precise
- Minimal

Never look like a generic SaaS dashboard.

---

## Core Philosophy

Every element must earn its existence.

Remove anything decorative.

Prioritize whitespace, typography, hierarchy, and motion.

The design should feel engineered rather than designed.

Think:

- Linear
- Vercel
- Stripe Dashboard
- Framer
- Notion Calendar
- Arc Browser

Avoid:

- Generic startup UI
- Large bordered cards
- Excessive shadows
- Marketing gradients
- Bright colors
- Template aesthetics

---

## Visual Language

### Background

Pure canvas.

Use subtle surface separation only when necessary.

Background:
#FFFFFF

#FAFAFA

Borders:
#EEEEEE

Primary text:
#111111

Secondary text:
#666666

Accent:
#2563EB

Only one accent color.

No rainbow palettes.

## Layout

Use an 8px spacing system.

Major sections:
120px vertical spacing.

Content width:
1200px max.

Generous whitespace everywhere.

Never center everything.

Prefer asymmetrical layouts.

---

## Navigation

Navigation should feel invisible.

Height:
56px

Minimal separators.

No boxed menu items.

Active states should be subtle:

- thin indicator
- soft background
- medium weight text

---

## Dashboard

The dashboard should feel like a command center.

Avoid large KPI cards.

Prefer:

- compact metrics
- clean data tables
- activity streams
- timelines
- command surfaces

Everything should fit naturally into a workflow.

---

## CMS

Design like Notion meets Airtable.

Large comfortable spacing.

Minimal controls.

Tables should feel lightweight.

Editing should happen in contextual panels.

Avoid modal-heavy workflows.

---

## Workflow Builder

This is the hero product.

Make it feel like a professional engineering tool.

Inspired by:

- Linear
- Figma
- Raycast
- Temporal
- GitHub Actions

Characteristics:

- dark canvas
- subtle grid
- thin connection lines
- elegant node cards
- zero visual noise

Nodes should feel like tools, not blocks.

Avoid:
- neon colors
- cartoon styling
- oversized controls

---

## Components

Buttons:
- 36–40px height
- 8px radius
- minimal shadows

Inputs:
- subtle borders
- quiet focus states

Cards:
- border first
- shadows only when necessary

Tables:
- primary data surface

Panels:
- lightweight
- contextual
- collapsible

---

## Motion

Motion should feel invisible.

Duration:

120ms
180ms
240ms

Use opacity and position.

Avoid:

- bounce
- elastic
- exaggerated scaling

Everything should feel immediate.

---

## Imagery

Avoid stock photos.

Use:

- product screenshots
- diagrams
- system illustrations
- abstract technical visuals

The product itself should be the hero.

---

## Quality Bar

If a screen looks like a template,
remove elements.

If a card can be removed,
remove it.

If a border can be removed,
remove it.

If a label can be removed,
remove it.

The final result should feel:

"Apple designed a developer operating system."

Minimal.
Precise.
Premium.
